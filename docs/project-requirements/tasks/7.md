# Task 7: Load products from static JSON file and create product utilities

**Dependencies:** None

### Goal
Load the products.json file into the application and create utility functions for accessing product data throughout the app.

### Implementation Context
**Files to Modify:**
- `public/products.json` (new - copy provided file)
- `src/shared/lib/products.ts` (new)

**Key Requirements:**
- Products.json contains 5 products with IDs, names, descriptions
- Default quantities: 10,000 for Pulse Pro High-Top, 5,000 for others
- Product data accessible throughout frontend and backend
- Type-safe product interfaces

**Technical Notes:**
- Place products.json in public/ directory for static file serving
- Create TypeScript interface for Product type
- Export utility functions: getProducts(), getProductById(), getDefaultQuantities()
- Use fetch or import to load JSON in frontend
- For backend (Convex), products can be imported directly

**Simplicity Decisions:**
- Static JSON file instead of database table (simpler for 5 products)
- No product CRUD operations (products are read-only)
- Hardcoded default quantities rather than configurable

### Scope Definition
**Deliverables:**
- products.json file in public/ directory
- Product TypeScript interface
- getProducts() utility function
- getProductById() utility function
- getDefaultQuantities() utility function returning default quote quantities

**Exclusions:**
- Product management UI (out of scope)
- Dynamic product pricing (out of scope)

### Implementation Steps
1. Copy provided products.json file to `public/products.json`
2. Create `src/shared/lib/products.ts` file
3. Define Product interface: `{ id: string, name: string, description: string, ... }`
4. Implement getProducts() function that loads and returns all products from JSON
5. Implement getProductById(id: string) function that finds product by ID
6. Implement getDefaultQuantities() function returning: `{ "pulse-pro-high-top": 10000, "other-product-1": 5000, ... }`
7. Export all interfaces and functions for use in components and backend
8. **Test: getProducts returns all 5 products**
   - **Setup:** products.json with 5 products
   - **Action:** Call getProducts()
   - **Expect:** Returns array of 5 product objects with correct structure
9. **Test: getDefaultQuantities returns correct values**
   - **Setup:** None
   - **Action:** Call getDefaultQuantities()
   - **Expect:** Returns object with Pulse Pro High-Top = 10000, others = 5000

### Success Criteria
- products.json successfully loaded in application
- Product interface provides type safety
- Utility functions return correct data
- Default quantities match requirements (10k, 5k, 5k, 5k, 5k)
- Functions work in both frontend and backend contexts

### Scope Constraint
Implement only product data loading and utilities. Do not implement product selection UI.
